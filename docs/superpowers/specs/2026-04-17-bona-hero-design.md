# bona-hero 影视感落地页设计规范 (Design Spec)

## 项目概述
**bona-hero** 是 bona 系列的全新落地页项目，旨在展示旗下的博卡、博乐、博文三款产品。核心目标是打造一个极具“电影感 (Cinematic)”的高端页面，实现从未登录状态的大视觉 Landing Page 到已登录状态下工作台的平滑过渡，保证用户的沉浸感不中断。

## 设计基调
- **核心风格：** 极致电影感 (Cinematic)、深色主题、极简巨幕排版。
- **视觉材质：** 噪点 (Noise)、渐变蒙版、暗角 (Vignette)、胶片质感。
- **展示产品：** 博卡 (图文/视频)、博乐 (一体化视频生成)、博文 (剧本创作)。

## 核心交互与状态定义

### 1. 首屏未登录状态（沉浸式浏览）
在此状态下，页面主要作为展示和引导转化（登录/注册）的窗口。

- **视觉布局：** 沉浸式全屏铺满（Edge-to-edge）。背景视频 100% 覆盖视口，巨型电影感文案和交互元素直接悬浮于视频上方，无多余边框。
- **交互模式：** Scroll Hijack (滚轮劫持) + 单次滚动强阻尼 (Snapping)。
  - 用户滚动滚轮不改变页面原生滚动条，仅触发产品的切换。
  - 单次滚动立即触发切镜（硬切或遮罩划像过渡），并伴有冷却时间（Debounce），防止连续触发导致画面混乱。
  - 右侧配合胶片感进度条（01/02/03）同步跳动指示当前展示产品。

### 2. 状态变形转场（State Morphing）
当用户完成登录鉴权，或已登录用户直接访问页面时触发的过渡动画。

- **视觉效果：** 电影遮幅式收缩 (Cinematic Letterboxing)。
  - 100vh 的全屏视频背景不像普通网页那样上卷，而是像电影荧幕拉下黑边遮幅一样，上下边缘平滑地向画面中段收拢，最终定格为高度约 30-40vh 的工作台顶部 Banner。
  - 此动画过程保持视频的原始比例不被压缩或拉伸，保留电影画面的构图感。
  - 收缩完成后，滚轮劫持解除，页面恢复正常的原生向下滚动能力。

### 3. 首屏已登录状态（项目库工作台）
转场完成后，Hero 区域变为顶部 Banner，下方平滑浮现用户内容。

- **Tab 切换：** 在 Banner 区域（或紧挨其下方）提供博卡、博乐、博文的 Tab 切换。切换时，Banner 的背景视频和下方数据列表同步进行电影感切镜过渡。
- **项目库瀑布流：** 采用悬浮胶片卡片 (Hover Film Strips) 设计。
  - 弱化或取消卡片的常规边框，以大面积的项目封面图为主体。
  - 默认状态带有轻微暗角；Hover 时封面轻微放大提亮，增加悬浮感，模拟胶片质感。
  - 点击卡片跳转至对应产品的详情页/编辑器。

## 技术栈与实现建议（供开发参考）
- **前端框架：** React / Next.js
- **动画引擎：** GSAP (ScrollTrigger / Observer) - 强力推荐使用 GSAP 处理复杂的 Scroll Hijack 和时间轴动画（Timeline），以保障 60fps 的流畅体验。
- **样式方案：** TailwindCSS，配合全局 CSS 变量控制底层电影材质（如噪点 SVG overlay）。
